# Hermes 原生接入改造计划（已实施）

**目标：** 使用现有 Hermes Telegram Gateway 和 `fat-loss-tracker` Skill，不再运行独立 Telegram Bot。

**架构：** Hermes 接收聊天消息后，通过 Skill 调用项目内 `backend/service.py`；JSONL 仍是唯一源数据。网页使用 Windows Python 或 WSL 启动，始终只读。

### 已实施

- `prompt.md` 更新为 Hermes 原生接入规则。
- 删除 `gateway/telegram_bot.py`、`.env.example` 和独立网关测试。
- 创建 Hermes 用户 Skill：`fat-loss-tracker`。
- `backend/service.py` 改用跨 Windows/WSL 项目锁。
- 新增 `启动减脂仪表盘.bat` 和 `停止减脂仪表盘.bat`。
- README 改为 Telegram 直接对现有 Hermes 说话的使用方式。
- 自动化测试覆盖跨平台锁、并发幂等、Windows 启动文件和独立 Bot 删除状态。
