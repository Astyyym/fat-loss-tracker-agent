# 减脂记录 Agent 当前版本需求文档

## 1. 产品定位

减脂记录 Agent 是一个本地优先、单用户的饮食、运动与身体测量记录工具。它通过 Hermes 已有消息入口或本地命令调用 `backend/service.py`，以 JSONL 事件保存历史，并通过 localhost 网页查看汇总和趋势。

当前版本只整理和稳定已经存在的能力，不开发 GitHub Issues 中规划的新功能。

## 2. 当前用户流程

1. 用户通过 Hermes 或命令行明确提出记录、查询、纠错、删除或恢复请求。
2. Hermes 路由 Skill 将原文交给 `backend/service.py`。
3. 后端完成校验、估算、幂等判断、事件追加和汇总。
4. 用户收到后端生成的 Markdown 结果。
5. 用户可在 Windows 双击启动本地只读仪表盘查看今日、最近七天和趋势。

## 3. 当前功能范围

- 记录饮食、运动、体重和腰围。
- 支持 correction、delete、restore 追加事件。
- `data/YYYY.jsonl` 是唯一可信业务数据。
- 支持日汇总、本周复盘、趋势和表格查询。
- 支持本地配置周期、热量目标、蛋白质目标与时区。
- 支持项目内锁、原子写入、备份和损坏数据报错。
- 提供 localhost 只读网页。
- 提供 Windows 一键启动与停止脚本。
- 附带公开、可移植的 Hermes 路由 Skill。

## 6. 环境与唯一正式版本

- 维护者以 Windows 中名为 `减脂记录Agent` 的项目目录作为唯一正式源码位置。
- Hermes 在 WSL 中通过 Windows 挂载路径访问同一目录，两种写法指向同一份文件。
- 不在 WSL home 或其他位置保留独立项目克隆。
- GitHub 保存远程版本历史，但不保存私人运行数据。

## 7. 数据与隐私边界

以下内容必须保持本地并由 Git 忽略：

- `config.json`
- `data/*.jsonl`
- `backups/**/*.jsonl`
- `summaries/*.md`
- `cards/*.html`
- `runtime/`
- `.env*`

公开代码、README、Prompt、Skill 和测试不得包含真实 Token、chat ID、个人绝对路径或真实健康记录。示例数值必须仅作虚构测试用途。

- Git 仓库、分支和远程保持完整。
- 本地 Hermes 路由参考更新到新路径。
- 自动化测试全部通过。

- 公开文件敏感信息扫描无真实泄漏。
- 服务可启动，`/` 与 `/api/dashboard` 返回成功；未完成的 Windows GUI 验收必须如实说明。

4. 目录迁移前检查引用，迁移后验证 Git、测试与服务。
